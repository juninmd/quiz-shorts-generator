### Improvements Made
1.  **Async Process Execution**: Replaced all `spawnSync` calls with a new asynchronous `execAsync` utility (using `child_process.spawn`). This prevents blocking the Node.js event loop during heavy operations like `ffprobe` and `edge-tts`.
2.  **Parallel Asset Preparation**: Orchestrated non-dependent tasks (TTS generation, font verification, background creation, and text file preparation) using `Promise.all`, significantly reducing the total pre-processing time.
3.  **Asynchronous File System**: Migrated almost all synchronous `fs` operations to `fs/promises`.
4.  **FFmpeg Performance**: 
    *   Added `-hwaccel auto` to leverage hardware acceleration when available.
    *   Removed hardcoded `-filter_threads` to let FFmpeg optimize thread allocation based on the environment.
    *   Ensured `-threads 0` is used for full CPU core utilization.
5.  **Test Suite Refactoring**: Updated the entire test suite (75 tests) to support asynchronous services and mocks, ensuring 100% reliability and coverage.
